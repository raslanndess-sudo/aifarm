# Definition of Done

Обязательное чтение перед началом любой задачи. Tester не закрывает задачу если критерии не выполнены. PM не диспатчит задачу если критерии не прописаны.

---

## 1. Все задачи (общее)

- [ ] `npx tsc --noEmit` — 0 ошибок
- [ ] Линтер чистый (`npm run lint` если настроен)
- [ ] Код написан согласно задаче, не больше и не меньше

---

## 2. Browser automation (Playwright / CDP)

- [ ] Селекторы взяты из **реального UI** — скриншот или dump селекторов в `docs/`
- [ ] **Live e2e тест** пройден в реальной целевой среде (не grep по коду)
- [ ] Screenshot или лог выполнения приложен к задаче (в `data/` или `docs/`)
- [ ] **Селекторы помеченные как "ПРИМЕРНЫЕ / ПРИБЛИЗИТЕЛЬНЫЕ / TODO" = automatic fail.** Задача не закрывается

---

## Разделение ответственности для browser automation

Backend-агенты работают в WSL, а Windows-Chrome управляется Playwright только из Windows — архитектурный факт. Поэтому live-тесты браузерной автоматизации распределены:

**Backend-агент:**
- Пишет код фичи
- Пишет test-скрипт (например scripts/task-007-live-test.ts)
- Прогоняет всё что доступно из WSL: `npx tsc --noEmit`, unit-тесты, code review
- Даёт пользователю: команду для запуска + чек-лист что проверять + структуру evidence-pack
- Публикует user-testing инструкцию в docs/task-N-user-testing.md

**Пользователь:**
- Запускает test-скрипт из Windows-PowerShell
- Снимает скриншоты в data/task-N-evidence/ по чек-листу из user-testing.md
- Возвращает evidence-pack

**Tester-агент:**
- Проверяет что evidence-pack соответствует чек-листу (по файлам, не запуская сам)
- Сверяет артефакты с критериями DoD
- Закрывает таск если всё совпадает

**Таск закрыт когда:**
- [ ] Код готов + tsc clean + unit-тесты pass (backend)
- [ ] Live тесты прогнаны пользователем, evidence-pack собран (user)
- [ ] Tester подтвердил соответствие (tester)

Это не снижение DoD — это уточнение кто физически нажимает Enter. Selector-hardening, fail-closed, запрет ПРИМЕРНЫХ — всё это остаётся в силе.

## 3. Внешние API (Kling, Higgsfield, Leonardo, будущие)

- [ ] Хотя бы один реальный вызов API отработал успешно
- [ ] Error handling протестирован: что будет при 500, 429, таймауте

---

## 4. UI (Frontend)

- [ ] Ручная проверка в браузере на localhost
- [ ] Скриншот финального состояния приложен

---

## 5. Рефакторинг / миграции

- [ ] Прогон существующих тестов
- [ ] Смоук-тест старого функционала — убедиться что не сломано

---

## Правило эскалации

Если критерии задачи **не содержат** пункта из этого документа, который **должен быть** для данного типа задачи — **tester и PM обязаны дописать критерии ДО начала работы**, не после.

PM проверяет критерии каждого таска по этому документу перед dispatch. Это зона ответственности PM.
