**Архитектура команды:**
```
┌────────────┬──────────────┐
│            │  Frontend    │
│            ├──────────────┤
│    PM      │  Backend     │
│  (human    ├──────────────┤
│   talks    │  Tester      │
```

**Правила команды:**
- PM не пишет код, только дистпатчит и верифицирует

- Агенты отчитываются через `say.sh` с типами: `status` / `done` / `blocked` / `ask` / `handoff`
- PM **обязан** перечитывать `tail -30 chat.md` перед выводами (научил отдельно — он ошибся один раз)

- Emergency Stop (красная кнопка) — админ-гейтед эндпоинт, убивает higgsfield-режим + audit log
- Non-admin юзеры гарантированно идут по Kling API пути (early return в resolveProvider перед DB-read)

### Блок 5 — Проблемы и их решения

| Проблема | Решение |
|---|---|
| PM в tmux не видит пастнутые скриншоты как `[Image #N]`, а только путь текстом | PM научен конвертить `E:\` → `/mnt/e/` и читать через Read tool; если `E:` съелось — добавлять сам |
| Скролл колёсиком → tmux в copy-mode → клавиатура не в ту панель | Научили работать; `q`/`Esc` в фокусную панель выходит |
| `(goto line) авы` внизу + escape-коды мыши в тексте | Нашли что это sub-режим copy-mode в chat-log панели; пофиксили `send-keys -X cancel` |
| `mouse on` автоматически переключает в copy-mode при скролле | Оставили включённым, юзер знает как выходить |
| Layout панелей ломался (backend не туда попадал) | Перешли на pane-IDs через `-P -F '#{pane_id}'` + `main-vertical` layout |
| PM ошибся в оценке статуса (сказал "Backend залип") | Научили: `tail -30 chat.md` ВСЕГДА перед выводами |
